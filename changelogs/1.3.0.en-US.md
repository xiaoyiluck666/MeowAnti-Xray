# Meow Anti-Xray 1.3.0 Changelog

This release updates Meow Anti-Xray for Minecraft 26.2, refreshes the loader dependencies from official sources, and adds new 26.2 stress-test results.

## Compatibility

- Supports Minecraft `26.2`; Java remains `25`.
- Fabric now targets Fabric Loader `0.19.3` and Fabric API `0.153.0+26.2`.
- NeoForge now targets `26.2.0.7-beta`.
- Fabric metadata now requires `>=26.2 <26.3`; NeoForge's Minecraft range is `[26.2,26.3)`.

## Load Runner

- `tools/mc-2612-load-runner.js` now supports `--mc-version 26.2`, using Minecraft 26.2 / protocol `776` by default.
- The previous `26.1.2` / protocol `775` profile is still available for regression checks.
- Fixed configuration-stage custom payload handling for 26.2 by no longer echoing server plugin messages back as serverbound payloads.

## Stress Test Results

These runs used the default `async-queue-size=16` with 8 fake network players driven by RCON spectator teleports:

| Loader | Duration | Chunk packets | Throughput | Client errors | Rewrite avg | Sync fallback |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| Fabric | `92s` | `32,893` | `357.49 chunks/s` | `0` | `2.839ms` | `42.0%` |
| NeoForge | `92s` | `35,593` | `386.86 chunks/s` | `0` | `2.719ms` | `39.0%` |

Seeing `pressure=saturated` during this synthetic exploration load is expected: the bounded async queue is applying backpressure to protect memory stability.

## Paper Parity

- Rechecked PaperMC/Paper `main`; the remote HEAD is still `76d2ac758cb3abe75aceefa88207443768f585c6`, matching this project's existing Paper anti-xray parity baseline.
- No new upstream Paper anti-xray defaults, hidden/replacement behavior, or permission-node changes were found, so no core anti-xray behavior change was needed for Paper parity.
