# Meow Anti-Xray 1.3.0 Minecraft 26.2 压测报告

日期：2026-06-27

## 结论

Meow Anti-Xray `1.3.0` 在 Minecraft `26.2` 下通过 Fabric 与 NeoForge 双 loader 启动、RCON 状态检查和 8 网络假玩家区块加载压测。两端压测均为 `0` 客户端错误，`/antixray profile` 显示默认背压队列按预期进入 `pressure=saturated`，未出现无界异步任务堆积。

PaperMC/Paper `main` 复查结果：远端 HEAD 仍为 `76d2ac758cb3abe75aceefa88207443768f585c6`，与项目既有 Paper anti-xray parity 基线一致；未发现新的 anti-xray 上游配置或行为需要对齐。

## 压测工具

工具：`tools/mc-2612-load-runner.js`

本轮升级点：

- 新增 `--mc-version 26.2`，默认使用 protocol `776`。
- 保留 `--mc-version 26.1.2`，用于旧版本回归。
- 修正 26.2 configuration 阶段 custom payload：忽略服务端插件消息，不再原样回发，避免服务端解码失败。

Fabric 命令：

```powershell
node tools\mc-2612-load-runner.js --mc-version 26.2 --clients 8 --duration 91 --connect-delay 100 --rcon-teleport --rcon-password meow-local-rcon-20260519 --gamemode spectator --teleport-interval 2500 --teleport-step 192
```

NeoForge 命令：

```powershell
node tools\mc-2612-load-runner.js --mc-version 26.2 --clients 8 --duration 91 --connect-delay 100 --port 25566 --rcon-teleport --rcon-port 25576 --rcon-password meow-local-rcon-20260519 --gamemode spectator --teleport-interval 2500 --teleport-step 192
```

## 风险与说明

- 本报告使用本地 Gradle dev server 和离线模式假客户端，适合验证协议兼容、区块包重写吞吐和背压行为，不等同于生产服务器长期在线压测。
- 默认 `async-queue-size=16` 仍以防止快照任务堆积和内存稳定为优先；内存更充足且跑图压力更高的服务器可尝试 `32` 或 `64`，并用 `/antixray profile` 观察 `syncFallbackRatio`。
- 本轮没有执行 Modrinth 上传。
